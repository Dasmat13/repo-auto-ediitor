The process

Five steps from URL
to fully understood

1
Clone the repository
Clones your GitHub repo into a secure temporary folder. Works with any public repo via HTTPS.
2
Scan for code files
Recursively walks the directory tree, detects every supported source file by extension, skips binaries, lock files, node_modules, and files over 100KB.
3
Annotate with AI
Each file is sent to your chosen AI provider with a carefully crafted prompt. Comments are added to every single line. Bugs are flagged inline. Code logic is never modified.
4
Generate study notes
For each file, a structured Markdown notes file is created — key concepts, must-know points, bug log, and at least 5 interview Q&As tailored to that file's content.
5
Push to GitHub
All changes are staged, committed with a descriptive message, and pushed back to your branch. Use --skip-push to annotate locally only.
